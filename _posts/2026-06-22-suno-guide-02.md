## プロンプトの基本構造

Sunoのプロンプト（Song Description欄）には、以下の**4要素**を組み合わせるのが基本だ。

```
[ジャンル], [ムード/雰囲気], [BPM/テンポ], [楽器編成or参照アーティスト]
```

**例：**
```
upbeat city pop, summer nostalgia, BPM 118, guitar + synth bass, female vocal, Japanese lyrics
```

この4要素をすべて入れることで、AIが解釈する余地が狭まり、イメージ通りの出力が安定する。

---

## 要素①：ジャンル指定

Sunoは数百種のジャンルを認識する。以下は特に反応が良いジャンルタグだ。

### J-POPと日本語音楽

| タグ | 生成傾向 |
|------|---------|
| `city pop` | 70〜80年代風シティポップ。松原みき・竹内まりや系 |
| `j-pop` | 現代的なJポップ。ストリングス入りバラード多め |
| `anime ost` | アニメサントラ風。ピアノ＋オーケストラ |
| `visual kei` | ヴィジュアル系。ディストーションギター＋ドラマチック展開 |
| `enka` | 演歌。哀愁のある歌謡曲風 |
| `vocaloid style` | ボカロP風。電子音＋高BPM |

### 洋楽ジャンル

| タグ | 生成傾向 |
|------|---------|
| `lo-fi hip hop` | 落ち着いたビート。BGM向き |
| `synthwave` | 80年代シンセサイザーの音 |
| `indie folk` | アコースティック。温かみある音色 |
| `jazz fusion` | インスト中心。複雑なコード進行 |
| `trap` | 重いハイハット＋808ベース |
| `bossa nova` | ブラジル音楽。クラシカルギター＋ソフトな歌声 |

---

## 要素②：ムード・雰囲気タグ

ジャンルだけでは足りない。感情・情景を加えると一気に精度が上がる。

**よく使うムードタグ：**

| 日本語イメージ | 英語タグ |
|------------|---------|
| 悲しい・切ない | `melancholic`, `bittersweet`, `sad` |
| 明るい・楽しい | `upbeat`, `cheerful`, `joyful` |
| 落ち着く・夜 | `relaxing`, `midnight`, `atmospheric` |
| 壮大・感動的 | `epic`, `cinematic`, `emotional` |
| 激しい・疾走感 | `energetic`, `powerful`, `intense` |
| 懐かしい・レトロ | `nostalgic`, `vintage`, `retro` |

**組み合わせ例：**
```
melancholic city pop, late night drive, neon lights, female vocal
```

---

## 要素③：BPMとテンポ指定

BPMを数値で指定すると、テンポのコントロール精度が上がる。

| 用途 | BPM目安 |
|------|---------|
| バラード・スローな曲 | 60〜80 |
| ポップス・標準的な曲 | 90〜120 |
| ダンスミュージック | 120〜140 |
| テクノ・ハードコア | 140〜180 |

**注意：** BPMを入れても必ずしも正確ではない。±10程度のズレは許容範囲として扱う。

---

## 要素④：楽器編成と参照アーティスト

### 楽器指定

よく効くタグ：
- `piano only` → ピアノソロ
- `guitar + bass + drums` → バンドサウンド
- `strings + choir` → オーケストラ風
- `synthesizer` → シンセ主体
- `acoustic guitar` → アコースティック

### 参照アーティスト（スタイル指定）

`in the style of [アーティスト名]` の形で入れると、そのアーティストのスタイルに近づく。

**有効な例：**
- `in the style of Kenshi Yonezu`
- `inspired by Aimer`
- `similar to Yoasobi`
- `lo-fi like Nujabes`

> **注意：** アーティスト名はそのままコピーされるわけではない。あくまでスタイルのヒント。著作権上の問題はないが、実際の楽曲に酷似することはほとんどない。

## プロンプトのNG例と改善例

### NG例①：短すぎる
```
❌ 明るい曲
```
```
✅ upbeat j-pop, sunny afternoon, BPM 128, piano + acoustic guitar, female vocal, Japanese lyrics
```

### NG例②：指示が矛盾している
```
❌ calm and energetic electronic music with heavy metal guitar
```
```
✅ calm electronic music with soft synth pads and subtle guitar
```

### NG例③：固有名詞を多用しすぎ
```
❌ Make a song like Taylor Swift meets Radiohead meets Metallica
```
```
✅ pop rock, emotional, introspective lyrics, distorted guitar with clean vocals
```

## 実践：プロンプト別生成結果の比較

同じ「夏の曲」でプロンプトを変えると全然違う曲になる：

**プロンプトA：**
```
summer song, Japanese
```
→ テンポも楽器もジャンルもバラバラ。毎回異なる

**プロンプトB：**
```
upbeat city pop, summer nostalgia, BPM 118, guitar + synth, female vocal, Japanese lyrics
```
→ 80年代シティポップ風の一貫した楽曲が安定生成される

---

## まとめ：プロンプトの黄金比

```
[ジャンル] + [ムード] + [BPM] + [楽器or参照アーティスト] + [言語指定]
```

この5要素を入れるだけで生成精度は劇的に上がる。次回（#03）では、Sunoの構造理解に必要な「**セクションタグ**」を解説する。イントロ・Aメロ・サビを意図的にコントロールする方法だ。
